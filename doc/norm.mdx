# ⛓ 规范

## 1. redux store

1. 全部采用符合范式化的数据集进行定义

## 2. 命名：

1. 使用 PascalCase 为类型命名。
2. 不要使用 I 做为接口名前缀，结尾用 face。
3. 使用 PascalCase 为枚举值命名。
4. 使用 camelCase 为函数命名。
5. 使用 camelCase 为属性或本地变量命名。
6. 不要为私有属性名添加\_前缀。
7. 尽可能使用完整的单词拼写命名。

## 3. 组件

1. 个文件对应一个逻辑组件 （比如：解析器，检查器）。
2. 共享的类型应该在 types.ts 里定义。
3. 在一个文件里，类型定义应该出现在顶部。

## 4. null 和 undefined：

1. 使用 undefined，不要使用 null。
